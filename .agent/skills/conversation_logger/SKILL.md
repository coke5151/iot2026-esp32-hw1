# Conversation Logger Skill

## 核心規則 (Core Rules)

在你（Antigravity）與使用者進行對話或處理請求時，**必須** 在任務完成前執行以下動作：

1. **總結對話內容**：將本次使用者的主要需求，以及你所做的所有規劃、修改的檔案清單、解決的技術細節等進行總結。總結可以使用條列式的大綱形式，但**請絕對不要減損重要的技術細節**（例如具體的設定值、修改了哪些邏輯等）。
2. **記錄到檔案**：將這份總結紀錄寫入到專案根目錄下的 `聊天記錄.md` 檔案中。
   - 專案根目錄：使用當下的 workspace root（例如 `c:\Users\user\Desktop\4112064214\iot2026-esp32-hw1`）。
   - 如果 `聊天記錄.md` 不存在，請主動建立它。
   - 請將新的對話紀錄「附加 (append)」到檔案中（你可以先讀取該檔案內容，再將新紀錄串接後寫回去）。

## 紀錄格式範例參考

請保持每次紀錄有清晰的標題與時間戳記：

```markdown
## [YYYY-MM-DD HH:MM] 任務紀錄

- **使用者請求：** (簡述使用者這次要你做什麼)
- **處理大綱與細節：** 
  - (列出執行的步驟 1)
  - (列出執行的步驟 2，保留遇到的問題與解決方案等細節)
- **修改的檔案：** (列出異動的檔案名稱)
```

## 注意事項
- 這是常駐型的行為方針，每次對話只要有實質進展都請確切執行。
